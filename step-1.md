## ЭТАП 0. Инициализация проекта

### Task 0.1 — Создание Django-проекта


**Cursor prompt:**


> Создай Django-проект с использованием Django Ninja.
>
> Архитектура проекта должна быть ориентирована на разные типы заданий ЕГЭ,
> где **каждый тип задания — отдельный Django app**.
>
> ### Общие требования
>
> * Все приложения должны находиться в папке `apps/`
> * Использовать SQLite как базу данных по умолчанию
> * Подключить Django Ninja и базовый API router
> * Проект должен запускаться сразу после создания
> * Код должен быть аккуратным, расширяемым и production-ready
>
> ### Обязательные apps
>
> Создай следующие Django apps:
>
> * `users` — кастомная модель пользователя
> * `attempts` — универсальные попытки решений
> * `core` — базовые абстракции и общие интерфейсы
> * `ai_engine` — LangChain, LLM, prompts, chains
>
> ### Task-ориентированные apps
>
> Для каждого типа задания создать отдельный app с префиксом `task_`
> (например: `task_truth_table`, `task_code_exec`).
>
> **ВНУТРИ КАЖДОГО `task_xxx` app строго использовать следующую структуру:**
>
> ```
> task_xxx/
> ├── models.py     # только модели, специфичные для этого типа задания
> ├── services.py   # чистая бизнес-логика и алгоритмы решения (без Django)
> ├── ai.py         # prompts и LangChain chains только для этого типа задания
> ├── views.py      # Django Ninja endpoints (тонкий слой)
> ├── admin.py
> ├── apps.py
> └── __init__.py
> ```
>
> ### Архитектурные правила (обязательные)
>
> * `views.py` **НЕ содержит бизнес-логики**
> * `services.py` **НЕ зависит от Django**
> * `ai.py` **НЕ содержит HTTP-логики**
> * Общие абстракции задач и попыток размещать в `core`
> * Связь `attempts` ↔ `task_xxx` реализовать через `GenericForeignKey`
>
> ### Settings
>
> * Вынести настройки в `config/settings/base.py` и `config/settings/dev.py`
> * Подключить все apps в `INSTALLED_APPS`
>
> ### Результат
>
> * Проект успешно запускается (`python manage.py runserver`)
> * Структура готова к добавлению новых типов задач без рефакторинга
