# 1. КРИТЕРИИ ВЫБОРА LLM (именно для твоего кейса)

Твоя задача — **не чат**, а **AI-репетитор по логике и алгоритмам**. Значит модель должна:

### 🔑 Критично важно

1. **Строго следовать инструкциям**
2. Уметь **пошагово рассуждать**
3. Возвращать **валидный JSON**
4. Хорошо работать с:

   * логикой
   * псевдокодом
   * Python
5. Быть **стабильной** (не галлюцинировать ответы ЕГЭ)

### ❌ Не важно

* креативное письмо
* «душевность»
* большие тексты

# 2. РЕКОМЕНДУЕМАЯ СХЕМА (BEST PRACTICE)

# 4. ПАРАМЕТРЫ МОДЕЛЕЙ (ОЧЕНЬ ВАЖНО)

## 4.1. Анализ решений (JSON, логика)

```python
LLM_ANALYZER = {
    "model": "gpt-4.1-mini",
    "temperature": 0.1,
    "top_p": 0.9,
    "max_tokens": 800,
    "presence_penalty": 0.0,
    "frequency_penalty": 0.0,
    "response_format": "json"
}
```

💡 Почему:

* `temperature 0.1` → почти нет фантазий
* JSON стабилен
* одинаковые решения → одинаковый вывод

---

## 4.2. Объяснения и обучение

```python
LLM_TUTOR = {
    "model": "claude-3.5-sonnet",
    "temperature": 0.4,
    "top_p": 0.95,
    "max_tokens": 1200
}
```

💡 Почему:

* допускаем вариативность объяснений
* «живой» язык
* не теряет логику

---

## 4.3. Генерация задач

```python
LLM_GENERATOR = {
    "model": "gpt-4.1-mini",
    "temperature": 0.6,
    "top_p": 0.95,
    "max_tokens": 1000
}
```

⚠️ После генерации **ОБЯЗАТЕЛЬНО**:

* прогонять через авто-решатель
* отбрасывать неоднозначные задачи

# 5. LangChain: КАК СВЯЗЫВАТЬ

## Рекомендуемая структура

```text
chains/
 ├── analyze_solution_chain.py
 ├── explain_thinking_chain.py
 ├── hint_chain.py
 ├── task_generator_chain.py
 └── answer_checker_chain.py
```

### Жёсткое правило

> **Одна chain = одна ответственность**

---

prompt = ChatPromptTemplate.from_messages([

# 6. ЗАЩИТА ОТ ГАЛЛЮЦИНАЦИЙ (CRITICAL)

Для образовательного продукта **обязательно**:

### ✅ Делай всегда

* строгие схемы JSON
* повторную валидацию
* fallback на авто-алгоритм
* логирование всех ответов LLM

### ❌ Никогда

* не доверяй LLM «правильный ответ»
* не принимай объяснение без проверки
* не генерируй финальный ответ без дедукции
